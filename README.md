# Matthew

**A local-first desktop app for semantic search and AI chat over your personal PDF library.**

My friend Matthew had accumulated over 10GB of PDF files — books, papers, articles he actually cared about. He wanted to have a real conversation with them, but every tool he tried hit a wall: file size limits, upload caps, paywalls. One day he asked me if I knew something that just worked. I looked around and could not find it, so I built it.

---

## What It Does

Matthew lets you build a personal knowledge base from PDF files and explore it through semantic search and AI-assisted chat.

- **Ingest** — drop in PDFs and Matthew chunks, embeds, and indexes them locally using your choice of embedding model
- **Search** — find semantically relevant passages across your entire library, not just keyword matches
- **Chat** — ask questions in natural language and get answers grounded in your documents, with source attribution for every reply
- **History** — every conversation is saved per-library and resumable across sessions, with sources preserved

**What runs locally:** PDF parsing, chunking, embedding generation, and vector search all happen on your machine. Your documents are never uploaded anywhere.

**What uses the internet:** AI chat responses are generated by [OpenRouter](https://openrouter.ai) — your query and the retrieved document passages are sent to the LLM you choose. Free models are available. The vector search itself remains local.

## Embedding Models

Matthew supports multiple local embedding models via [fastembed](https://github.com/Anush008/fastembed-rs). You choose a model per library — it's downloaded once and cached locally in `~/.cache/huggingface/hub/`.

| Model | Dimensions | Size | Best for |
|---|---|---|---|
| `BAAI/bge-m3` | 1024 | ~580MB | Multilingual, general purpose (default) |
| `mixedbread-ai/mxbai-embed-large-v1` | 1024 | ~560MB | English, high accuracy |
| `nomic-ai/nomic-embed-text-v1.5` | 768 | ~270MB | English, fast |
| `intfloat/multilingual-e5-large` | 1024 | ~560MB | Multilingual |
| `BAAI/bge-base-en-v1.5` | 768 | ~220MB | English, lightweight |
| `sentence-transformers/all-MiniLM-L6-v2` | 384 | ~90MB | Very fast, lower accuracy |

The embedding model is locked per library after the first import — switching models on existing libraries would make vectors incompatible.

## Getting Started

### Prerequisites

- [Rust](https://rustup.rs/) (stable)
- [Node.js](https://nodejs.org/) 18+
- [Tauri v2 prerequisites](https://tauri.app/start/prerequisites/)

> **Platform:** Matthew is developed and tested on Linux (Pop!_OS / Ubuntu). Tauri supports macOS and Windows in principle, but these platforms have not been tested.

### Run in Development

```bash
git clone https://github.com/story-essentia/matthew.git
cd matthew
npm install
npm run tauri dev
```

On first launch, Matthew downloads your selected embedding model. This only happens once per model — subsequent launches are fully offline for search.

### Build for Production

```bash
npm run tauri build
```

Generates an `.AppImage` and `.deb` package in `src-tauri/target/release/bundle/`.

## Configuration

Matthew stores settings in `~/.config/matthew/settings.json`:

```json
{
  "apiKey": "your-openrouter-api-key",
  "model": "openrouter/free"
}
```

An [OpenRouter](https://openrouter.ai/keys) API key is required for chat. Free models are available — `openrouter/free` auto-selects the best one at no cost. You can also use paid models like Claude, GPT-4o, or Gemini.

Libraries and their vector databases are stored wherever you choose on your filesystem. Chat history is saved alongside each library in a `chats/` subfolder.

## Features

- **Multi-model embeddings** — choose your embedding model per library from six supported options
- **Chunk presets** — Precise (512 chars), Balanced (2048), or Contextual (4096) chunking, locked after first import
- **Semantic search** — cosine similarity search with configurable result count
- **RAG chat** — retrieves relevant passages before each LLM call, conversation memory preserved within sessions
- **Source attribution** — every AI reply links to the exact document passages used; click to inspect them
- **Chat history** — auto-saved per library, resumable with full source preservation
- **Deduplication** — SHA-256 hashing prevents re-importing the same document twice
- **Adaptive batch sizing** — memory-safe embedding based on available RAM
